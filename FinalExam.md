### 2. GIT, VERSIONAMIENTO Y TRABAJO PROFESIONAL
#### 4. Explica con tus palabras el flujo básico de trabajo en Git (commit, push, pull, branch, merge).

* Esto depende del paso en el que estemos. Tendremos que comenzar bajando la última versión del trabajo para modificarla.
  1. Primero se hará el documento el cual subir y usar el commit para indicar las modificaciones del documento.
  2. Después se creará, modificará o eliminará alguna rama en la cual se colocará el documento.
  3. Para poder ver todas en una sola rama se utilizará merge para combinar las ramas, y estas se pueden combinar 2 o más ramas.
  4. Se utilizará push para subir el archivo deseado.
