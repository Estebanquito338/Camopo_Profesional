# Comandos Basicos De La Terminal
# - pwd - Muestra el directorio actual en el que te encuentras.
# - ls - Lista los archivos y carpetas en el directorio actual.
# - ls -l - Lista con más detalles.
# - ls -a - Lista todos los archivos, incluidos los ocultos.
# - cd <ruta> - Cambia de directorio.
# - cd .. - Sube un nivel (al directorio padre).
# - cd ~ - Te lleva al directorio home.
# - mkdir <nombre_carpeta> - Crea una nueva carpeta.
# - touch <nombre_archivo> - Crea un nuevo archivo vacío.
# - cp <origen> <destino> - Copia un archivo o carpeta.
# - mv <origen> <destino> - Mueve o renombra un archivo o carpeta.
# - rm <nombre_archivo> - Elimina un archivo.
# - rm -r <nombre_carpeta> - Elimina una carpeta y su contenido (¡Cuidado!).
# - cat <nombre_archivo> - Muestra el contenido de un archivo.
# - grep <patrón> <archivo> - Busca un texto dentro de un archivo.

# Comandos Básicos De GitHub
# - git config --global user.name "Tu Nombre"
# - git config --global user.email "tu@email.com"
# - Flujo Básico de Trabajo
# - git init - Convierte una carpeta local en un repositorio de Git.
# - git clone <url_del_repositorio> - Descarga un repositorio remoto (de GitHub) a tu máquina.
# - git status - Muestra el estado de tus archivos (qué está modificado, qué está listo para commit, etc.).
# - git add <nombre_archivo> - Añade un archivo al "área de preparación" (staging).
# - git add . - Añade todos los archivos modificados.
# - git commit -m "Mensaje descriptivo del cambio" - Guarda los cambios del "área de preparación" en el historial con un mensaje.
# - git push - Sube tus commits locales al repositorio remoto (ej. a GitHub).
# - git pull - Descarga los últimos cambios del repositorio remoto y los fusiona con tu versión local.
